The drift

Every source becomes a special case.

Getting data out of one API once is easy. Doing it the same way across many sources, keeping it reliable in production, and not minting a new one-off every time is the hard part.

Without a spine

  • × One-off scripts only one person trusts
  • × Different auth, retry & pagination per source
  • × Config scattered across code, notebooks, and glue
  • × Pipelines that break on backfills and new destinations

With Spine

  • → Pipelines defined in YAML, validated up front
  • → One service / parser / loader shape every time
  • → A real execution plan with dependency resolution
  • → Retries, backfills & failure isolation per resource
What it is

A runtime, not a connector marketplace.

Modular by design: clear extension points for services, handlers, parsers, collectors, and loaders — swap a part without a rewrite.

Configuration-driven

YAML-based sources, resources, defaults, and queries. Define pipelines; don't hand-write them.

Modular architecture

Clear seams for services, handlers, parsers, collectors, and loaders — swap parts without a rewrite.

Execution planning

Dependency resolution and optimized ordering, so staged runs happen in the right sequence.

Resilient execution

Validation up front, retries, backfills, and failure isolation per source and resource.

Extensible design

Add sources, auth methods, loaders, and transformations without inventing a new flow.

Flexible inputs

Path, query, body, and dynamic parameters — dates, upstream data, and correlated inputs.

The config contract

Describe the pipeline. Spine runs it.

A source, its resources, and where they load — declared once, validated before anything runs. The same shape whether it's a REST API or a relational database.

Read the config guide
config/sources/jsonplaceholder.yml 
source:
  name: jsonplaceholder
  type: rest
  base_url: https://jsonplaceholder.typicode.com
  auth: none

resources:
  - name: posts
    path: /posts
    retries: 3
    loader:
      destination: local
      format: delta
      write_mode: merge
      # validated before the run starts
The execution model

One flow, every source.

Define

Sources, resources, defaults, and queries as YAML under config/.

Validate

Config shape and runtime assumptions are checked up front — fail early, not mid-run.

Plan

Dependencies resolve into a staged execution plan with the right order.

Run

The same service, parser, collector, and loader patterns extract, transform, and load — every time.
